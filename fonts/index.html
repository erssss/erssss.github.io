    Liberator: 一个用于 GPU 超额显存订阅的数据重用框架 | 三作


  • 如今，图数据的大小迅速增加，往往会超过 GPU 显存，需要频繁的 GPU 与 CPU 数据交换。我们深入分析了不同粒度图应用的内存访问模式，提出了 Liberator ，使得 GPU 显存被保留给可能跨迭代轮次访问的数据，以避免主内存和 GPU 之间过多的数据传输。对于不存在于 GPU 内存中的数据，采用了合并与对齐的内存访问方式来提高传输效率，同时通过并行处理 GPU 内存中的数据和主内存中的数据来进一步优化。实验结果表明，Liberator 与最先进的方法相比实现了平均2.7倍的速度提升。

    基于 oneAPI 的大规模图计算异构加速框架设计 | 队长


  • 异构计算需要在 GPU 与 CPU 之间进行频繁的数据搬运，GPU 长时间处于忙等待状态，极大地影响了程序性能。针对此，我们发现图计算过程中应用程序总在一个较大的访存范围内是对图数据进行顺序的扫描。基于这一发现，我们通过划分 GPU 显存来分别存储可复用数据及其他数据，以优化程序的访存行为，并基于 oneAPI 提出了一个通用的图计算框架。
